# AnnoMI Client Talk-Type Classification

Privacy-preserving classification of client utterances in Motivational Interviewing
(MI) therapy transcripts into three talk types — **change**, **neutral**, and
**sustain** — for use as the perception layer of a robot-assisted physical-therapy
adherence pipeline.

---

## Headline Result

The best system is **Qwen3-Embedding-8B + LoRA** (rank-16 adapters on
attention projections, ~0.3% of parameters trainable, class-weighted
cross-entropy loss).

| Metric | Value |
|---|---|
| Test macro F1 | **0.6131** |
| Test accuracy | **73.1%** |
| Sustain (minority) F1 | **0.431** |
| GPT-4o-mini few-shot reference | 0.6019 |
| MiniLM frozen baseline | 0.5465 |

Our local 8B model matches GPT-4o-mini few-shot performance on the same test set
while running entirely on local hardware — supporting the project's central
privacy motivation. Full per-class metrics, the 22-experiment leaderboard, and
the confusion matrix live in
[`EXPERIMENTS_REPORT.md`](EXPERIMENTS_REPORT.md).

## Project Overview

### Task

Three-class classification of single client utterances into MI talk types:

- **Change** — language toward behavior change (*"I want to start walking again."*)
- **Sustain** — language against behavior change (*"I just don't have time."*)
- **Neutral** — neither

### Dataset

[AnnoMI](https://github.com/uccollab/AnnoMI) — 133 motivational interviewing
transcripts, 9,699 utterances total, **4,817** labeled client utterances after
filtering. Splits are **transcript-disjoint** (stratified by transcript-level MI
quality) to prevent leakage:

| Split | N | change | neutral | sustain |
|---|---|---|---|---|
| Train | 3,153 | 23% | 66% | 11% |
| Val | 798 | 25% | 67% | 8% |
| Test | 866 | 25% | 66% | 9% |

### Approach

Sentence encoder → last-token pool → MLP classifier (2-layer, dropout, class-weighted CE).
Across 22 experiments we sweep three axes:

1. **Encoder scale** — MiniLM (22M) → MPNet → RoBERTa → DeBERTa-v3 → Qwen3-Embedding-8B.
2. **Training regime** — frozen encoder + MLP, end-to-end fine-tune, LoRA fine-tune.
3. **Auxiliary signals** — synthetic data augmentation, conversational context windows,
   GPT-4o-mini prompt classification (zero-shot and few-shot).

Primary metric: **macro F1** (weights all three classes equally despite imbalance).

## Environment Setup

Create the conda environment:

```bash
conda env create -f environment.yml
conda activate annomi-mlp
pip install -e .
```

Or with pip directly:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -e .
```

GPU is required for the Qwen-8B experiments (24 GB VRAM minimum, e.g. AWS
`ml.g5.2xlarge` / RTX 4090). Smaller-encoder experiments run comfortably on
8–16 GB GPUs and most run on CPU.

## Reproduction

### Stage 1 — preprocessing + TF-IDF baseline

```bash
python -m annomi_pipeline.scripts.run_stage1 --config configs/data_config.yaml
```

Produces `data/processed/{train,val,test}.jsonl`,
`data/tokenized/{train,val,test}.jsonl`, and the TF-IDF baseline JSON / PNG in
`data/outputs/`.

### Stage 2 — frozen-encoder + MLP experiments

```bash
# Compute embeddings
python -m annomi_pipeline.scripts.run_embeddings --config configs/embeddings_config.yaml

# Train MLP head
python -m annomi_pipeline.scripts.run_stage2 --config configs/train_config.yaml
```

Swap the config files for any of the encoder/context/augmentation variants
listed in `configs/`.

### Stage 2 — fine-tuning experiments

```bash
python -m annomi_pipeline.scripts.run_finetune --config configs/train_config_roberta_finetune.yaml
```

### Phase 2 — Qwen-8B + LoRA (the winner)

The Qwen-8B experiments were run as standalone scripts on AWS:

```bash
# Frozen Qwen-8B + MLP head
python notebooks/qwen8b_embedding_aws.py

# LoRA fine-tune (rank 16 on attention projections)
python notebooks/qwen8b_lora_aws.py
```

Outputs land in `data/outputs_qwen8b_lora/{metrics.json,confusion_matrix.json}`.

### GPT-4o-mini reference experiments

```bash
export OPENAI_API_KEY=...
python scripts/run_gpt4o_mini_classify.py --mode zero_shot
python scripts/run_gpt4o_mini_classify.py --mode few_shot --k 3
```

These are reference benchmarks only — not deployable systems, since they violate
the project's privacy constraint.

### Regenerate figures

```bash
python scripts/make_report_figures.py
```

Reads `data/outputs_qwen8b_lora{,_aug}/{metrics.json,confusion_matrix.json}`
and writes seven PNGs to `figures/`.

## Key Findings

- **Scale + gradient flow is the unlock.** Frozen 8B plateaus; LoRA adapters
  on attention projections move sustain F1 from 0.379 → 0.431, the first
  system in the project to break 0.40 on the minority class.
- **Synthetic augmentation regressed across all three regimes** (s200, s400,
  s800). The o4-mini-generated paraphrases produced an aggressive sustain
  prior (recall 0.40 → 0.64, precision 0.46 → 0.33), but the distribution did
  not match real AnnoMI sustain talk.
- **Conversational context did not help** with mean-pooled MiniLM (test F1
  monotonically regressed: ctx0=0.547 → ctx4=0.455). Last-token pooling on
  decoder-style encoders (Qwen) is the natural follow-up.
- **The local 8B model matches GPT-4o-mini few-shot** (0.613 vs 0.602),
  validating the privacy-preserving deployment path.

See [`EXPERIMENTS_REPORT.md`](EXPERIMENTS_REPORT.md) for the full leaderboard,
training trajectories, and confusion matrices.

## Future Work

Apply the lab's prior **Guacamole + knowledge-distillation** recipe (a QLoRA
4-bit compression pipeline that retained 91% of ChatGPT-3.5 quality on OASST1
when applied to Llama-3-8B) to the Qwen-8B + LoRA winner. Target: shrink VRAM
from ~16 GB (fp16) down to ~5 GB while preserving macro F1 ≥ 0.60, enabling
deployment on consumer/edge hardware (e.g., Apple M-series 24 GB unified
memory) for the HARP Lab robot PT-coach pipeline.
